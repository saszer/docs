## Smart Filters & Global Date Range

### Using Smart Filters

The Smart Filters system provides powerful filtering capabilities to narrow down transactions for viewing and analysis.

<Steps>
  <Step title="Access Smart Filters">
    Click the expandable "Smart Filters" card at the top of the All Transactions page.
  </Step>
  
  <Step title="Choose Filter Type">
    Use tabs to switch between Quick Mode (preset filters) and Advanced Mode (custom filters).
  </Step>
  
  <Step title="Apply Filters">
    Filters apply automatically as you change them. Press Enter in search fields for instant refresh.
  </Step>
  
  <Step title="View Active Filters">
    Active filter count is displayed on the Smart Filters header. Clear all with the reset button.
  </Step>
</Steps>

### Filter Categories

<AccordionGroup>
  <Accordion title="Date & Time Filters">
    **Date Range Options:**
    - **Quick Presets**: Today, This Week, This Month, Last Month, This Year, Last Year
    - **Month Presets**: January through December (for specific month filtering)
    - **Custom Range**: Select specific start and end dates
    - **All Time**: View all transactions without date limits
    
    **How it Works:**
    - Date filters apply to transaction date (not upload date)
    - Presets automatically calculate date ranges
    - Filters persist across page refreshes

  <Accordion title="Search & Text Filters">

## AI-Powered Analysis Features

The All Transactions page includes three powerful AI analysis buttons that work with your current filter state to provide intelligent insights and automation.

### 1. Smart Categorize

AI-powered bulk categorization that automatically assigns categories to multiple transactions at once.

<Card title="Access Smart Categorize" icon="brain" href="https://app.ai2fin.com/all-transactions" horizontal>
Located at the top of the All Transactions page, this button analyzes all filtered transactions for intelligent categorization.
</Card>

<Steps>
  <Step title="Apply Filters (Optional)">
    Use Smart Filters to narrow down transactions you want to categorize (e.g., "This Month" + "Uncategorized").
  </Step>
  
  <Step title="Click Smart Categorize Button">
    Click the "Smart Categorize" button to start AI analysis of filtered transactions.
  </Step>
  
  <Step title="Review AI Suggestions">
    Modal opens showing AI-suggested categories for each transaction with confidence scores.
  </Step>
  
  <Step title="Apply or Modify">
    Accept all suggestions, modify individual ones, or skip transactions. Click "Apply Categories" to save.
  </Step>
</Steps>

<AccordionGroup>
  <Accordion title="How Smart Categorize Works">
    **AI Analysis Process:**
    - Analyzes transaction descriptions and merchant names
    - Matches against historical categorization patterns
    - Uses machine learning for intelligent suggestions
    - Provides confidence scores for each suggestion
    
    **Features:**
    - Works with filtered transaction sets
    - Bulk categorization for efficiency
    - Manual override capability
    - Learning from user corrections
  </Accordion>
  
  <Accordion title="Best Practices">
    **Tips for Effective Use:**
    - Start with uncategorized transactions filter
    - Review high-confidence suggestions (90%+)
    - Correct low-confidence suggestions to train AI
    - Use date filters to categorize by period
    - Combine with "This Month" for regular maintenance
    
    **When to Use:**
    - After CSV import with new transactions
    - Monthly categorization maintenance
    - Bulk processing of similar transactions
    - Initial setup of transaction history
  </Accordion>
</AccordionGroup>

### 2. Pattern Analysis

Automatically detect recurring bills and payment patterns from your transaction data.

<Card title="Learn More About Patterns" icon="repeat" href="/features/pattern-analysis" horizontal>
Pattern Analysis uses AI to identify recurring transactions and create automatic bill tracking.
</Card>

<Steps>
  <Step title="Set Date Range">
    Use Smart Filters to set an appropriate date range (e.g., "Last 6 Months" for better pattern detection).
  </Step>
  
  <Step title="Click Pattern Analysis">
    Click the "Pattern Analysis" button to start analyzing transaction patterns.
  </Step>
  
  <Step title="Review Detected Patterns">
    Modal shows all detected recurring patterns with frequency, amount, and merchant details.
  </Step>
  
  <Step title="Create Bill Patterns">
    Select patterns to convert into recurring bill tracking. AI will monitor future occurrences.
  </Step>
</Steps>

<AccordionGroup>
  <Accordion title="Pattern Detection Capabilities">
    **What Gets Detected:**
    - **Monthly Bills**: Subscriptions, utilities, rent, loan payments
    - **Frequency Patterns**: Weekly, monthly, quarterly, annual
    - **Amount Consistency**: Fixed or variable amounts
    - **Merchant Matching**: Same merchant recurring transactions
    - **Date Patterns**: Specific dates or date ranges
    
    **Pattern Types:**
    - Fixed amount recurring (exact match)
    - Variable amount recurring (range-based)
    - Same-day recurring (specific date)
    - Approximate date recurring (within range)
  </Accordion>
  
  <Accordion title="Filter Integration">
    **Works Best With:**
    - **Date Range**: 3-12 months for accurate detection
    - **Category Filter**: Focus on specific expense types
    - **Amount Range**: Find patterns in specific price ranges
    - **Merchant Filter**: Analyze specific vendor patterns
    
    **Use Cases:**
    - Identify all subscription services
    - Track monthly utility bills
    - Monitor loan and credit payments
    - Detect seasonal expenses
  </Accordion>
  
  <Accordion title="Pattern Management">
    **After Creation:**
    - Patterns appear in "Patterns (Recurring)" sub-page
    - Future matching transactions auto-flagged
    - Edit pattern rules and parameters
    - Pause or delete patterns as needed
    - Export pattern reports
    
    **Pattern Monitoring:**
    - Alerts for missed patterns
    - Amount variance notifications
    - Date shift warnings
    - New occurrence confirmations
  </Accordion>
</AccordionGroup>
